\# API Test Cases



\## テスト対象



https://httpbin.org



\## テストケース一覧



| No | 分類 | テスト名 | 操作 | 期待結果 |

|---|---|---|---|---|

| 1 | 正常系 | GET APIステータス確認 | `/get` にGETリクエストを送信 | ステータスコード200が返る |

| 2 | 正常系 | GET API JSON確認 | `/get` にGETリクエストを送信 | レスポンスJSONに `url` が含まれる |

| 3 | 正常系 | POST API JSON確認 | `/post` にJSONをPOST送信 | 送信した `name` と `job` が返る |

| 4 | 異常系 | 404確認 | `/status/404` にGETリクエストを送信 | ステータスコード404が返る |

| 5 | 異常系 | 500確認 | `/status/500` にGETリクエストを送信 | ステータスコード500が返る |

| 6 | 効率化 | parametrize 200確認 | `/status/200` にGETリクエストを送信 | ステータスコード200が返る |

| 7 | 効率化 | parametrize 404確認 | `/status/404` にGETリクエストを送信 | ステータスコード404が返る |

| 8 | 効率化 | parametrize 500確認 | `/status/500` にGETリクエストを送信 | ステータスコード500が返る |



\## 観点



\- 正常系のステータスコード確認

\- JSONレスポンスの中身確認

\- POST送信データの確認

\- 404/500の異常系確認

\- parametrizeによる複数パターン確認
